Componentes vulnerables y obsoletos

Es probable que sea vulnerable si: 
Si no conocen las versiones con las que trabajan y acumula una cantidad de componentes del lado del cliente como tambien del servidor.
Si los desarrolladores de software no comprueban la compatibilidad de las bibliotecas actualizadas, mejoradas o parcheadas.
 Si el software es vulnerable, no tiene soporte o no está actualizado. 
 Esto incluye el sistema operativo, el servidor web de aplicaciones, el sistema de gestión de bases de datos, las aplicaciones, las API y todos los componentes, los entornos de ejecución y las bibliotecas.
Cómo prevenir

Debe existir un proceso de gestión de parches para:
Eliminar dependencias no utilizadas, características innecesarias, componentes, archivos y documentación.
Vigile las bibliotecas y componentes que no reciben mantenimiento o no crean parches de seguridad para versiones anteriores. Si no es posible aplicar parches, considere desplegar un parche virtual para supervisar, detectar o proteger contra el problema descubierto.
Sólo obtenga componentes de fuentes oficiales a través de enlaces seguros.
